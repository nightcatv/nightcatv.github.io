What is Flask?

  • 使用Python寫的輕量級Web應用框架。
  • 通常被稱為 “microframework(微框架)”，原因：
    • 使用簡單的核心
    • 用extension增加其他功能
  • 沒有預設的資料庫、表單驗證工具 => 保留了擴增的彈性
  • Flask是基於下列工具所開發出來的：
    • WSGI
      • Web Server Gate Interface
      • 為Python網頁應用程式開發的標準，主要為介於網路應用程式與伺服器之間的規範
    • Werkzeug
      • WSGI的工具
      • 允許網頁框架以此為基礎來建立
    • Jinja2
      • Python的模板引擎 (Templating Engine)

Flask和Django的差別：

  • 兩個之間，Django就像是套裝電腦、Flask就像是組裝電腦一樣的差別
    • Django：
      • 有自己的 authentication system 以及 Admin 管理系統
      • 有自己的 ORM 以及 migration system
      • 有自己的 templating language
      • manage.py 也幫你準備好了
    • Flask：
      • extension 很多，很多事情找到對的 extension 掛上去就好了
      • 使用 Jinja2 作為 templating language
      • 定義 URL 很簡單

Route & view Function

  • 網頁瀏覽器的用戶端會將request送到網頁伺服器，伺服器在將其送到Flask app實例
    • Flask app實例必須知道那些程式處理用戶端請求的每一個URL => 有個URL到Python函式的對應
    • URL與處理它的函式之間關係稱為路由(route)
  • 定義路由的方式
    • 最方便的是使用app實例公開的app.route裝飾器：

      裝飾器 => Python語言的標準功能，經常用來將函式註冊成在某些事件發生時呼叫的處理函式。
      想了解更多可參考Python Decorator 入門教學

    • 較為傳統的方式則使用app.add_url_rule()，基本形式接收三個參數：URL、端點名稱、view函式
  • 如index()這類處理app URL的函式便稱為view函式
    • 若app在www.example.com上部屬，則index()會在瀏覽器前往https://www.example.com/時執行
    • view函式回傳值即是response，是在瀏覽器視窗顯示的文件

一般來說在Python源碼中放HTML程式碼會讓程式難以維護，此僅作示範用。

  • 另外也可以用一種帶有動態成分的方式來寫路由：
    • 角括號 => URL動態的部分，所有符合固定部分的URL都會對應到這個路由
    • 動態路由預設使用字串，但也可使用不同型態，例：/user/<int:id>，支援的型態有：
      • string
      • int
      • float
      • path => 特殊字串型態，可使用正斜線

Template

  • 商業邏輯(business logic)：收到含使用者資料的request，Flask將其指派給處理該類request的view函式
  • 展示邏輯(presentation logic)：該view函式必須與資料庫溝通已進行動作，產生一個response回傳給伺服器
  • 若將商業邏輯、展示邏輯混在一塊 => 程式碼難以理解、維護
  • 模板是個含有 回應文字 及 保留給動態部分變數 的檔案
    • 動態部分僅在request的context裡面才能知道
    • 將這些變數改為實際值並回傳最終回應字串的程式為render
    • 使用Jinja2模板引擎來轉譯模板
