SQL and NoSQL Database

SQL Database

  • 關聯式資料庫：
    • 將資料存入資料表
    • 用資料表建立實體(Entity)的模型
  • 資料表含：
    • 固定的欄位 (Column)

      定義資料表所有代表實體的資料屬性

      • 有一個主鍵 (primary key)，保存一個不重複的識別碼 => 辨識每一列
      • 可能有外鍵 (foreign key)，參考同一表或另一表裡某列的主鍵 => 之間的連結稱為關聯 (relationship)
    • 可變數量的資料列 (Row)

      每一列都定義一個實際資料元素，並將值指派某些/所有欄位

NoSQL Database

  • 經常使用集合 (collection) 來取代資料表，文件 來取代紀錄
  • 設計上較難聯結，故大多不提供這種操作 => 若要查詢需要自行執行聯結操作
  • 資料表較少 => 資料會重複
  • 重複的資料可加快查詢速度，因為不需要聯結

SQL or NoSQL?

  • SQL優點：用高效、緊湊的形式來儲存結構化資料
    • 資料盡量保持一致性
    • 具備ACID
      • Atomicity (不可分割性)
      • Consistency (一致性)
      • Isolation (隔離性)
      • Durability (持久性)
  • NoSQL對ACID要求不高，故有時會有性能的優勢
  • 對中小型而言，SQL和NoSQL效能差不多

Python Database Framework

  • 多數資料庫引擎都有Python套件，包括開放原始碼、商業套件
  • 另外亦可用資料庫抽象層套件：SQLAlchemy、MongoEngine

    抽象層：物件關係對應器(ORM) / 物件文件對應器(ODM)

    • 可在較高層面上使用正規的Python物件
  • 選擇資料庫框架應評估下列因素：
    • 易用性
    • 效能
    • 可攜性
    • Flask整合

Use Flask-SQLAlchemy

  • SQLAlchemy => 一種強大的關聯式資料庫框架，支援多資料庫後端

  • 首先進行安裝

    • 在SQLAlchemy，資料庫是用URL指定
    資料庫引擎 URL
    MySQL mysql://username:password@hostname/database
    Postgres postgresql://username:password@hostname/database
    SQLite (Linux, MacOS) sqlite:////absolute/path/to/database
    SQLite (Windows) sqlite:///c:/absolute/path/to/database
    • URL中：
      • hostname：承載資料庫服務的伺服器
      • database：想使用的資料庫名稱
      • username、password：資料庫使用者憑證

      SQLite沒有伺服器，故沒有hostname、username、password，database是資料庫在磁碟中的檔名

    • 用SQLALCHEMY_DATABASE_URI來設置app資料庫的URL；將SQLALCHEMY_TRACK_MODIFICATIONS設為False (使用較少的記憶體)

Model Definition

  • 在ORM環境中，Model通常是Python的類別 => 其屬性匹配對應的資料表欄位
  • SQLAlchemy資料庫實例提供模型基礎類別、一組用來定義他們結構的協助類別與函式
    • 使用roles、users定義Role、User模型
    • __tablename__：定義資料庫的資料表名稱

      若省略，SQLAlchemy會指派預設表明

    • db.Column：定義模型屬性
      • 第一引數 => 資料庫欄位型態、模型屬性
      • 其他引述 => 指定各屬性設置選項
      • 常見欄位型態
      型態名稱 Python型態 說明
      Integer int 一般的整數，通常32 bits
      SmallInteger int 短範圍整數，通常16 bits
      BigInteger int / long 不限精度整數
      Float float 浮點數
      Numeric decimal.Decimal 定點數
      String str 可變長度字串
      Text str 可變長度字串，經最佳化，供極長 / 不限長度的字串使用
      Unicode unicode 可變長度Unicode字串
      UnicodeText unicode 可變長度Unicode字串，經最佳化，供極長 / 不限長度的字串使用
      Boolean bool 布林值
      Date datetime.date 日期值
      Time datetime.time 時間值
      Datetime datetime.datetime 日期與時間值
      Interval datetime.timedelta 時間間隔
      Enum str 字串值串列
      PickleType Any Python Object 自動Pickle序列化
      LargeBinary str 二進位blob
      • 常見欄位選項
      選項名稱 說明
      primary_key 設為True => 該欄位就是表的主鍵
      unique 設為True => 該欄位不允許重複值
      index 設為True => 該欄位建立索引，讓查詢指令更有效率
      nullable 設為True => 該欄位允許空值(null)
      default 定義該欄位預設值
    • __repr__()：提供給人看得字串

      非必要，有助於除錯

Relation

  • 藉由關係建立不同表兩列間的連結
    • e.g. 角色對應使用者為一對多關係 => 一個角色可以指派給多個使用者、一個使用者僅一個角色
    • User模型中role_id欄位定義為外鍵
      • db.ForeignKey()的roles.id => 這個欄位存有roles表資料列的id值
    • Role模型中users：
      • 可用此屬性取得該角色相關使用者串列
      • db.relationship()
        • 第一個引數 => 對方模型
        • backref引數 => 將role屬性加入User模型 => 定義關係的反方向
        • 大多能自行找關係的外鍵位置，但有兩個以上欄位被定義成外鍵則須提供額外的引數
      • 常見關係選項
      選項名稱 說明
      backref 在關係的另一個模型加入反向參考
      primaryjoin 明確指令兩模型間的連結條件 (僅有不明確關係時需要)
      lazy 指定如何載入相關項目，可用值：select、immediate、joined、subquery、noload、dynamic
      uselist 設為False => 使用純量而非串列
      order_by 指定關係中的項目順序
      secondary 指令多對多關係使用的關聯表名稱
      secondaryjoin 指定多對多關係的二級連結條件

    項目 說明

    • 常見查詢指令方法
    項目 說明

    first() 回傳第一筆查詢結果，若無則回傳None

    get_or_404() 回傳匹配指定主鍵的資料，若無則終止請求並回應404
    count() 回傳查詢結果數量
    paginate() 回傳Pagination物件，內含指定範圍的結果

Migrate Database with Flask-Migrate

  • Flask-SQLAlchemy只會在資料表不存在時才會用模型建立
    • 更新資料庫唯一的方法：銷毀舊的資料表，同時也移除所有資料
  • 較好的做法：使用資料庫遷移框架
    • 可以追蹤資料庫結構描述的變動 => 如此套用漸進式更改
  • 方法有二：
    • SQLAlchemy的開發者寫了一個遷移框架 => Alembic
    • Flask-Migrate => 輕量的Alembic包裝器，其與flask命令整合一起

Build Migrate Script

  • Alembic可將資料庫設成變動紀錄中的任何時間點
  • 資料庫遷移是用遷移腳本來表示，函式：
    • upgrade() => 套用在遷移過程改變資料庫的部分
    • downgrade() => 會移除它們
  • 建立Alembic遷移分手動和自動
    • revision => 手動建立遷移，用空的遷移函式建立骨架腳本
    • migrate => 自動遷移，必須用Operations物件公開的指令來實作，會試著尋找模型的定義與目前資料庫狀況間的差異

      不一定準確，可能會遺漏一些不明確的細節

  • 為改變資料庫結構，需採取下列程序：
    • 對模型類別做必要的變更
    • 用flask db migrate建立自動遷移腳本
    • 檢查生成的腳本並修改
    • 將遷移腳本加入原始檔控制系統
    • 用flask db upgrade將遷移套用到資料庫

Upgrade Database

  • 接收遷移腳本後，可使用flask db upgrade套用到資料庫

    初次遷移相當於db.create_all()，之後只會對資料表套用變更
