Data Structures in JavaScript: Objects and Arrays

  • Number、Boolean、String皆為資料結構的元素之一
    • 許多型別資訊需要不只單一的因素
  • Object => 允許包含複數數值 (包括其他Object)
  • Javascript程式寫到目前可以發現到大多運算都是簡單的資料類型，故於此會簡單介紹資料結構。

The Weresquirrel

在大約晚上八點到十點之間，Jacques發現他會變形成一隻有著毛茸茸尾巴的囓齒動物，也就是松鼠。 一方面來說，Jacques很慶幸自己不如傳統的變形者，比起變成狼，變成松鼠所造成的影響較小，比起擔心一個不小心吃掉鄰居(這很令人為難)，他更擔心被鄰居的貓吃掉。兩個小時後，Jacques全身赤裸地在橡樹頂端的細枝上醒來且迷失了方向，他開始養成晚上鎖門窗並在地上放幾顆核桃的習慣。 如此能夠處理貓跟樹的問題，但Jacques更傾向完全不在意自身狀況，但這不尋常的變身情況讓他懷疑可能是某些事情引起的，有一陣子他認為這只會在他接近橡樹的幾天會發生，但遠離橡樹並未改善。 Jacques決定改以科學的方法，他開始紀錄每天狀況，無論變形於否，如此希望能夠限縮引發變形的情況。

Data Sets

  • 為了使用整串資料，需要找到方法在記憶體中表現它；比如說：為了表現2、3、5、7、11的數字集合可以有下列方法：
    1. string
      • string不限制長度，但在做數字運算需要進行型態轉換
      • Output: “2 3 5 7 11”
    2. array
      • 寫法：用方括號包在數值兩側，中間以逗點隔開，並從最左方的元素開始賦予index
  • array首個index從零開始 => listOfNumbers[0]

Properties

  • myString.length、Math.max => 用於檢視property (屬性)的表示式

    • 幾乎所有Javascript的值都有屬性，除了null、undefined，若嘗試去檢視這些非值的屬性會回傳錯誤

  • 檢視屬性的兩個符號：

    1. 點 dot
    2. 方括弧 square brackets

  • value.x、value[x]都是檢視value的屬性，但不一定是同個屬性

    • 主要看x被怎麼闡釋
      • 若使用. => 其後的x為屬性名稱
      • 若使用[] => 括號中的x被轉換成屬性名稱
    • value.x會回傳value的屬性x
    • value[x]會試著評估x這個表示式再使用其結果，轉換成字串後當作屬性名稱使用

  • 檢視屬性的三種方法：

    • 所有的屬性名稱都是字串，但可為任何字串
    • .後面只會對應有效的屬性名稱
    • 若想找其他屬性名稱像是2、Ore wa tensai desu則必須使用[]，如value[2]、value["Ore wa tensai desu"]

  • 陣列的元素作為陣列的屬性來儲存，並用數字來當屬性名稱

  • 陣列的其中一個屬性length會回傳元素的數量，因為我們已經知道這是一個有效的屬性名稱，故會寫array.length這種比較簡單的表示法來代替`array[“length”]

Methods

  • 字串、陣列的物件，其大多屬性都具function特性

  • 每個字串都有toUpperCase屬性 => 呼叫時回傳全轉為大寫字母的複製字串

    另外也有toLowerCase

    • 有趣的是我們並沒有傳任何的argument，但卻能夠取得"Doh"字串，這個概念會在之後做詳細介紹

  • 包含function的屬性通常稱為屬於該物件的method

    e.g. "toUpperCase is a method of a string"

  • 下面利用array來示範兩個methods：

    • push() => 新增值到陣列末
    • pop() => 移除陣列最後一個值並回傳給使用者
    • 這兩個名詞源自資料結構的一個概念 - stack

Objects

  • 看回到weresquirrel，一組日記記錄可以陣列呈現

    • 紀錄不可能只包含數字或字串 => 每組紀錄需儲存數項活動、Jacques變成松鼠的狀態布林值
    • 要把這些單獨的值群組化，並將其放入紀錄的陣列

  • object的值皆任意蒐集的屬性，新增一個物件的方法便是使用大括弧來表示

    • 大括弧中用逗點分隔屬性
    • 每個屬性皆以<propertyName>: <value>的形式
    • 使用複數行來寫物件 => 方便閱讀
    • 若是要讀取不存在的屬性 => 回傳undefined
    • 可使用=來賦予屬性的值：
      • 若是已存在的屬性會覆蓋其原值
      • 若是不存在的屬性則會新創一個屬性並賦予其值

  • 若物件中的屬性非一般有效名稱或是有效數字則須引號

  • 大括弧在Javascript中有兩個意義：

    1. 位在程式敘述部分的起始位置 => 代表「開始程式敘述」
      • 鮮少使用
    2. 在其他位置 => 代表「物件」

  • delete運算子

    • 為單元運算子
    • 會移除物件中已命名的屬性
    • 不常使用
    • in => 應用在字串或物件，會回傳是否有該屬性名稱存在

  • 設定屬性值為undefined跟實際移除該屬性的差別在於：

    • 前者的物件仍然存在該屬性，只是不存在該有的值
    • 後者則是不復存在該屬性，使用in會回傳false

  • Object.keys => 查看物件屬性，會回傳物件屬性名稱的陣列字串

  • Object.assign => 複製某個物件的所有屬性到另一個物件上

    • 總共兩個參數，第一個為目標物件，第二個為來源物件

  • 陣列即是一種特別用來儲存連續性資料的物件

    • 可以將其視為以數字做為屬性的物件

Multability

  • 我們理解到物件的值是可以修改的，然而值的型別卻是immutable (不可改變的)

  • 物件的運作方式較為不同，我們可以改變其屬性，使單一物件可含有不同的內容

  • 假設我們有兩個數字分別為120、120，雖然這兩個數字可能指向不同的記憶體位址

    • 以物件來說，兩個物件指向相同位址跟兩個具有相同屬性的物件，兩者是不同的
    • object1、object2兩者指向同一位址，故其中一個變動，另一個也會變；但object3指向不同位址，雖然起始值相同，但實際上卻有所區別
    • ==可用於比較物件
      • 回傳true => 兩物件精確擁有相同值
      • Javascipt中沒有更深入的比較，比如比較物件內容，但可由自己實現

  • Binding是可變動或為常數的

    • let => 可變動值
    • const => 若以物件，可能變動

    • 以pizza為例，其表格可能如下

      !pizza pizza
      !squirrel 76 9
      squirrel 4 1

    • 計算ψ的公式
      $$
      φ = {n_{11}n_{00} - n_{10}n_{01} \over \sqrt{n_{1ｘ} * n_{0ｘ} * n_{ｘ1} * n_{ｘ0}}}.
      $$

    • 計算過程 =>

      • 1 * 76 - 4 * 9 = 40
      • 5 * 85 * 10 * 80 = 340000
      • φ = 40 / sqrt{340000} = 0.069

    • 如此可以得知吃pizza不會是影響變形的主要因素

Further Arrayology

  • 接著會介紹幾個常用的、且好用的array method

  • shift、unshift

    • 這是管理任務的queue
    • getTask() => 取得第一個項目並從queue中移除
    • rememberUrgently => 增加任務到queue的第一個項目

  • indexOf、lastIndexOf

    • indexOf => 從頭開始找特定值的index
    • lastIndexOf => 從末端開始找特定值的index
    • 皆有第二個參數可輸入 => 從第幾個開始找

  • slice

    • 取起始以及終止的index，回傳其中的元素
    • 包含起始的index，不包含終止的index
    • 亦可省略終止index已取得起始index之後的所有元素

  • concat

    • 將特定陣列串聯成新的陣列，類似+在字串中的使用
    • 如果沒有concat而取走中間值，會分裂成兩個陣列 (若為一元陣列)

Strings and Their Properties

  • 若是常是添加新的屬性進到string中，並不會成功

    • string、number、Boolean這類型別並非物件，故無法新增屬性，但卻有內建好的屬性

  • slice、indexOf

    • 與array的method相似
    • 不同的點在於indexOf可以在字串中尋找一個以上的字元；陣列只能找單一個元素

  • trim

    • 移除字串首尾的空白、換行、tabs等等類似的字元

  • padStart

    • 在設計的長度內填充字元

  • split、join

  • repeat

  • length

Rest Parameters

  • 這在接收數字的argument非常方便，比如在Math.max可以接收所有argument

  • 寫法為在function最後一個參數前寫三個點

    • 此便稱為 rest parameter
    • 另外也能用陣列的argument呼叫function

  • 可以在兩個參數中間加上陣列，如此得出新的陣列

The Math Object

  • 與數字相關的實用function，其自己具備命名空間

    • 過多的全域變數命名會汙染命名空間，提高複寫的可能性

  • 三角函數

    • cos(cosine)、sin(sine)、tan(tangent)、acos、asin、atan
    • Math.PI => π的近似值

  • Math.random => 回傳新的介於0到1之間亂數值

  • Math.floor => 無條件捨去

    • 其他還有類似的如：
      • Math.ceil => 無條件進位
      • Math.round => 四捨五入
      • Math.abs => 取絕對值
