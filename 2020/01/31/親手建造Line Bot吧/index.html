Step 1. 申請Line Developers的帳號

  • 這一步其實非常簡單，首先到Line Developers的網站上，首先登入自己的Line帳號

  • 接著在Provider的位置點擊 Create，然後輸入名字

  • 然後點進Provider頁面後，由於本次主要做的是個訊息回覆機器人，所以點擊Message API來進行一連串的設定

  • 設定好之後，應該會有下列的畫面

  • 等等會有兩個值會需要使用到：

    • Basic Setting裡的Channel Secret
    • Message API裡的Channel access token

Step 2. Line Bot事件的元件

  • Line Bot的事件最常使用大約有五個要素：
    • Callback URL
    • Web Hook
    • Reply Message
    • Push Message
    • Profile

Callback URL

  • 這邊較麻煩，因為這邊需要提供Line Bot回傳的網址 => 需要簡易的Server處理Line Server傳來的資訊
    • 這個URL只支持HTTP！！很重要
    • 而且憑證要是有問題也不會接受

Web Hook

  • 發生事件都會發出這個Web Hook，同時程式也會收到event的JSON檔
  • 這JSON非常重要：
    • userId 紀錄使用Bot的使用者
    • groupId 紀錄被加到哪個群組
    • replyToken 可不管ID直接回覆給呼叫的人
    • message 可看到對方傳的指令
    • etc…
  • 可以到官方的文件看詳細的內容

Reply Message

  • 使用replyToken這個key裡的資訊來回覆訊息
    • 這樣就不用使用到userId

Step 3. 架設回應的伺服器

使用的環境與套件：

  • Pipenv
  • Flask
  • Line Bot api

架設Pipenv

  • pipenv是目前官方推薦的套件管理工具，主要提供下列特點：

    推薦個鬼，如果功能沒有用到那麼多的話，根本就超級難用的

    • 只需要pipenv，無需分別使用pip、virtualenv
    • 自動產生與更新Pipfile、Pipfile.lock，解決維護requirement.txt的問題
    • 透過套件Hash安全性檢查確認
    • 可產生dependency樹狀圖 ($ pipenv graph)
    • 可透過.env自動載入不同環境變數
  • 安裝pipenv的方法有二：
    • 透過pip安裝
    • 透過Homebrew安裝 (如果是MacOS的話)
  • 首先創立一個專案目錄，並且在目錄下創建虛擬環境
  • 接著安裝套件，目前以requests的套件作為示範
    • 若目錄下沒有Pipfile，則會自動產生一個，同時產生帶有hash資訊的Pipfile.lock
    • 若想移除套件
  • 要在虛擬環境中下指令方法有兩種：
    • 在不進入虛擬環境下，根據virtualenv執行指令
    • 進入虛擬環境
      • 若要離開虛擬環境

架設Flask

  • 首先在pipenv中安裝Flask的套件
  • 然後在專案資料夾裡建立一個app.py
  • 架設Flask非常的簡單，僅僅三個步驟：
    • Import & Setting
    • Add View
    • Run Server

Import & Setting

  • 這裡簡單宣告一個Flask的app實例
    • 接著如果需要任何設定都可在app.config調整

      app.config內的設定參數可參考網站官方文件

Add View

  • 在Flask裡，可使用app.route的裝飾器(decorator)來為路徑設定服務的function

Run Server

  • Server的基本設定已經完成，在Flask的套件裡啟動的方法都差不多，都是用app.run(host, post, ...)
  • 接著直接用python運行
    • 基本上Flask的Server都會預設在本機端的port 5000，所以就算沒有設定app.run()的狀態也會在127.0.0.1:5000上運行

Step 4. 結合Flask Server與Line Bot api

  • 先是要安裝Line Bot相關的套件
  • 在開始整合之前，要將環境進行設定
    • 新增一個.env檔案
    • 到Line Developers剛剛設立的Message API中找到兩個值
      • Basic Setting裡的Channel Secret
      • Message API裡的Channel access token
    • 將這兩個值寫入.env檔案中
  • 接著就按照先前所述的Flask架設的三個步驟一一來執行吧～

    這邊要將app.py全部重新寫過

Import & Setting

  • line_bot_api => 負責與Line本身的API做溝通
  • handler => 負責處理送過來的資料

Add View

  • 先處理handler
    • 利用這個handler.add來包裝自己處理line回傳的json資料

      在官方的line bot api response有完整json資料

    • MessageEvent => 收到文字訊息時啟動
  • 接著處理一個接收json的api
    • handler.handle(body, signature) => 判斷要用剛剛設定的哪個event來回應
    • InvalidSignatureError => Line sdk 會檢查資料是否正確(用CHANNEL_ACCESS_TOKEN)

Run Server

  • 一樣先將Flask運行起來
  • 為了讓Line有辦法在本地端先進行測試，我們使用ngrok這項工具來產生網址
    • 利用ngrok產生網址
    • 將得到的網址中將https的那段網址送到Line Developers的Webhook URL (在Messaging API中)
      • 接著打開"Use Webhook"
  • 如此利用手機掃描Messaging API中的二維碼加入好友，便能夠得到一個回聲機器人

Step 6. 將Line Bot發佈至Heroku

  • 先到Heroku的頁面點Create New App

  • 然後下載Heroku CLI
  • 透過Command Line進行Heroku的登入
  • 到Line Bot的目錄中設定git
  • 接著將Line Bot發佈到Heroku上
    • 之後若是需要進行專案更新便以這些指令來進行
  • 回到Heroku頁面Setting，找到Domain的部分，將URL加到Line Developer的Webhook部分便可以透過Line連上了
